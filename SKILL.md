Read the README.md file within this skill, and explore the other available files (colors_and_type.css, the assets/ folder, fonts/).

If creating visual artifacts (slides, mocks, throwaway prototypes, posters, social cards, etc.), copy assets out and create static HTML files for the user to view. Always link `colors_and_type.css` and use the CSS variables — never invent new colors. The three-color discipline (Void / Signal / Concrete + Bone for print) is non-negotiable.

If working on production code, you can copy assets and read the rules here to become an expert in designing with this brand. Use the approved lockups (Signal-on-Void, Bone-on-Void, Void-on-Signal, Signal-on-Bone) and avoid the forbidden ones (Signal-on-Concrete, pure #000, neon greens, cool blues, gradient orange).

The Mozaka wordmark is a logo, not a typeface. Never re-vectorize it or substitute Monument Extended for it.

If the user invokes this skill without any other guidance, ask them what they want to build or design (poster, deck, social drop, merch mockup, event ID, fan-club page, etc.), ask a few questions about the surface and audience, and act as an expert designer who outputs HTML artifacts *or* production code, depending on the need. Default to dark (Void) surfaces. Default voice: confident, industrial, rooted, mythic, founder-direct. No emoji. No exclamation marks. Uppercase for display, sentence case for body.
